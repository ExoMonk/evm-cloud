# Local Dev Stack

Boot a complete evm-cloud pipeline on your laptop using [kind](https://kind.sigs.k8s.io/) (Kubernetes in Docker).

---

## What this unlocks

- **Contract development** against a real indexer — deploy to Anvil, see events in ClickHouse
- **Config iteration** without cloud costs — tune rindexer.yaml, see results locally
- **Integration testing** before deploying to k3s/EC2/EKS

## Architecture

```text
┌─────────────────────────────────────────────────────────────────┐
│                     kind cluster (local)                         │
│                                                                  │
│  ┌──────────┐     ┌──────────┐     ┌──────────┐                │
│  │  Anvil   │────►│  eRPC    │────►│ rindexer │                │
│  │ :8545    │     │ :4000    │     │ :18080   │                │
│  │ (sim)    │     │ (proxy)  │     │ (indexer)│                │
│  └──────────┘     └──────────┘     └─────┬────┘                │
│                                          │                      │
│                                    ┌─────▼─────┐               │
│                                    │ ClickHouse │               │
│                                    │ :8123      │               │
│                                    └────────────┘               │
└─────────────────────────────────────────────────────────────────┘
```

All services are exposed via NodePort — no `kubectl port-forward` needed. Endpoints persist through laptop sleep.

## Requirements

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) with **4GB+ memory** allocated
- [kind](https://kind.sigs.k8s.io/docs/user/quick-start/#installation)
- [kubectl](https://kubernetes.io/docs/tasks/tools/)
- [Helm](https://helm.sh/docs/intro/install/)

## Quick start

```bash
# Boot the stack (~2 minutes, forks Ethereum mainnet by default)
evm-cloud local up

# Check all services
evm-cloud local status

# Tear down
evm-cloud local down
```

Or via Make targets:

```bash
make local-up
make local-status
make local-down
```

## Endpoints

| Service    | URL                        | Purpose                     |
|------------|----------------------------|-----------------------------|
| Anvil      | `http://localhost:8545`    | EVM simulator (JSON-RPC)    |
| eRPC       | `http://localhost:4000`    | RPC proxy (same as prod)    |
| ClickHouse | `http://localhost:8123`    | Query indexed data (HTTP)   |
| rindexer   | `http://localhost:18080`   | Indexer metrics             |

---

## Fork mode (default)

By default, the local stack forks Ethereum mainnet via `https://ethereum-rpc.publicnode.com`. Anvil serves real mainnet state with local block production. rindexer indexes real contracts.

To fork a different chain:

```bash
evm-cloud local up --rpc https://polygon-rpc.com
```

The CLI probes the chain ID from the RPC URL and configures all services accordingly.

:::warning
In fork mode, set `start_block` in your `rindexer.yaml` close to the fork block number. Historical `eth_getLogs` requests are proxied to the fork RPC, which may rate-limit or timeout on large ranges.
:::

## Mainnet mode

Index a live chain directly — no Anvil, eRPC points straight at your RPC:

```bash
evm-cloud local up --mainnet --rpc https://polygon-rpc.com
```

The stack deploys ClickHouse + eRPC + rindexer (no Anvil pod). eRPC uses the provided RPC URL as its upstream. This is useful for indexing real on-chain data locally without forking.

## Fresh mode

For pure local development without external dependencies:

```bash
evm-cloud local up --fresh
```

This uses chain_id=31337 (Anvil default) with no fork.

## Contract deploy hook

Run a script after the stack boots:

```bash
evm-cloud local up --post-deploy ./my-deploy.sh
```

The script receives environment variables:

```bash
ANVIL_RPC_URL=http://localhost:8545
ERPC_URL=http://localhost:4000
CLICKHOUSE_URL=http://localhost:8123
CHAIN_ID=1  # or whatever chain was probed
```

Example workflow:

```bash
#!/usr/bin/env bash
# Deploy contract
forge script script/Deploy.s.sol --rpc-url $ANVIL_RPC_URL --broadcast

# Update rindexer config with deployed address, then re-run:
# evm-cloud local up (idempotent Helm upgrade)
```

## Resource profiles

### `default` — laptop-friendly (~2.5Gi total)

For basic contract development. Fits on 8GB+ laptops with other apps running.

| Service    | CPU Req | Memory Req | CPU Limit | Memory Limit |
|------------|---------|------------|-----------|--------------|
| Anvil      | 100m    | 128Mi      | 500m      | 256Mi        |
| eRPC       | 100m    | 128Mi      | 250m      | 256Mi        |
| ClickHouse | 200m    | 512Mi      | 500m      | 1Gi          |
| rindexer   | 200m    | 256Mi      | 500m      | 512Mi        |

### `heavy` — fork mode (~8Gi total)

For fork-mode Anvil with large state. Requires Docker Desktop with 8GB+ memory.

```bash
evm-cloud local up --profile heavy --rpc https://eth.llamarpc.com
```

## Data persistence

By default, data is **ephemeral** — lost on `evm-cloud local down`, including rindexer progress tables. This means a full re-index from `start_block` on each restart.

For persistent sessions:

```bash
evm-cloud local up --persist
```

Data is stored in `~/.evm-cloud/local-data/`. To reset:

```bash
evm-cloud local reset
```

:::note
Persistent data does not survive Docker Desktop VM resets on macOS (kind runs inside the VM).
:::

## Config

Provide your own `rindexer.yaml` to index custom contracts:

```bash
evm-cloud local up --config-dir ./config
```

Place ABI files in an `abis/` directory next to the config file. If no config is provided, a starter config with a placeholder contract is generated automatically.

Re-running `evm-cloud local up` performs an idempotent Helm upgrade — no tear-down needed.

## All options

```bash
evm-cloud local up [OPTIONS]

  --rpc <url>            RPC URL (fork source, or direct upstream with --mainnet)
  --mainnet              Skip Anvil, point eRPC directly at --rpc URL
  --fresh                No fork, chain_id=31337
  --profile <name>       Resource profile: default, heavy (default: default)
  --persist              Persistent ClickHouse data across restarts
  --force                Force-recreate cluster even if it exists
  --config-dir <path>    Path to config directory or rindexer.yaml
  --post-deploy <path>   Script to run after stack is healthy
```

## Troubleshooting

### Port conflicts

`evm-cloud local up` checks if ports 8545, 4000, 8123, 18080 are available and reports which one conflicts. Free the port and retry.

### Docker memory

Increase in Docker Desktop → Settings → Resources → Memory. Minimum 4GB for default profile, 8GB for heavy.

### rindexer CrashLoop

Usually means it can't connect to ClickHouse. Check logs:

```bash
kubectl logs deploy/local-indexer --context kind-evm-cloud-local
```

### Cluster unreachable

NodePort mappings survive laptop sleep. If the cluster is gone (Docker restart), run `evm-cloud local up` — it recreates automatically.
