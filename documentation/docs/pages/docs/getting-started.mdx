**Fast path:**
- Configure variables and secrets (`secrets.auto.tfvars` + `minimal_clickhouse.tfvars`)
- Deploy with `evm-cloud deploy` (or raw Terraform)
- Verify containers are running on the EC2 host

### CLI Preview

<GettingStartedCliShowcase />

:::tip[Key Terms]
- **eRPC** — Open-source RPC proxy that aggregates multiple blockchain RPC endpoints with failover, caching, and load balancing
- **rindexer** — EVM event indexer that reads on-chain data and stores it in your database (ClickHouse or PostgreSQL)
- **BYODB** — Bring Your Own Database: you provide an existing database endpoint (e.g. ClickHouse Cloud), evm-cloud connects to it
- **cloud-init** — EC2 startup script that installs Docker and configures the instance on first boot
:::

### AWS Credentials

Configure AWS CLI with credentials that have EC2, VPC, IAM, and Secrets Manager permissions:

```bash
aws configure
# Or export environment variables:
export AWS_ACCESS_KEY_ID=your-key
export AWS_SECRET_ACCESS_KEY=your-secret
export AWS_DEFAULT_REGION=us-east-1
```

### SSH Key

If you don't have one:

```bash
ssh-keygen -t ed25519 -C "evm-cloud" -f ~/.ssh/evm-cloud
# Public key: ~/.ssh/evm-cloud.pub
# Private key: ~/.ssh/evm-cloud
```

## Your First Deployment

We'll use the `minimal_aws_byo_clickhouse` example — EC2 instance running eRPC + rindexer with an external ClickHouse database.

### 1. Initialize the example

```bash
mkdir base_example && cd base_example
evm-cloud init --example ec2_clickhouse 
```

Optional: list available presets from repo examples:

```bash
evm-cloud init --list-examples
```

### 2. Configure Secrets

```bash
cp secrets.auto.tfvars.example secrets.auto.tfvars
```

Edit `secrets.auto.tfvars` with your values:

```hcl
# SSH keys
ssh_public_key              = "ssh-ed25519 AAAA... your-email"
ssh_private_key_path        = "~/.ssh/evm-cloud"

# ClickHouse connection (from ClickHouse Cloud or your own instance)
indexer_clickhouse_url      = "https://your-instance.clickhouse.cloud:8443"
indexer_clickhouse_password = "your-password"
```

> **Don't have ClickHouse?** Sign up for [ClickHouse Cloud](https://clickhouse.cloud/) free tier — create a service, copy the HTTPS endpoint (e.g. `https://abc123.us-east-1.aws.clickhouse.cloud:8443`) and the password. Or use the `minimal_aws_rds` example instead (uses managed PostgreSQL, no external DB needed).

### 3. Preview

:::code-group

```bash [CLI]
evm-cloud deploy --dry-run
```

```bash [Terraform]
terraform plan -var-file=minimal_clickhouse.tfvars
```

:::

`evm-cloud deploy --dry-run` prints a concise summary and writes full Terraform plan details to `.evm-cloud/logs/terraform-plan-<timestamp>.log`.

Plan output includes resources for:
- VPC, subnets, security groups
- EC2 instance (t3.micro — AWS free tier eligible)
- IAM role + Secrets Manager secret
- Docker Compose services (eRPC + rindexer)

### 4. Deploy

:::code-group

```bash [CLI]
evm-cloud deploy
```

```bash [Terraform]
terraform apply -var-file=minimal_clickhouse.tfvars
```

:::

`evm-cloud deploy` prompts `Apply these changes? (y/N)` in interactive shells. In CI/non-interactive shells, use `evm-cloud deploy --auto-approve`.

This takes 3-5 minutes. Terraform will:
1. Create a VPC with public subnets
2. Launch an EC2 instance with Docker pre-installed
3. Deploy eRPC (RPC proxy) and rindexer (indexer) as Docker containers
4. Wire rindexer to your ClickHouse database via Secrets Manager

### 5. Verify

After `evm-cloud deploy` completes, the CLI prints a post-deploy summary with the instance IP and SSH command. Use the IP from that output:

```bash
# SSH into the instance (IP shown in post-deploy summary)
ssh -i ~/.ssh/evm-cloud ec2-user@<public-ip>

# Check containers are running
sudo docker compose -f /opt/evm-cloud/docker-compose.yml ps
```

You should see `erpc` and `rindexer` containers in `running` state.

```bash
# Verify data is flowing — check rindexer logs
sudo docker compose -f /opt/evm-cloud/docker-compose.yml logs rindexer --tail=20
# Look for "Indexed block ..." or "Stored N events" messages
```

### 6. Teardown

:::code-group

```bash [CLI]
evm-cloud destroy --yes
```

```bash [Terraform]
terraform destroy -var-file=minimal_clickhouse.tfvars
```

:::

For CI/non-interactive usage, run `evm-cloud destroy --yes --auto-approve`.
